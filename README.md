# 福彩3D 走势图（纯静态网站）

一个手机端为主的福彩 3D 走势图，**纯前端、无后端**，可直接部署到 GitHub Pages 并绑定自己的域名。

## 功能

- **百位 / 十位 / 个位**：标准走势图，红球为当期中奖号，灰色数字为“遗漏”期数，红线连接各期。
- **和值（0–27）/ 跨度（0–9）**：同样的走势 + 遗漏 + 连线。
- **形态**：和值、跨度、组三/组六/豹子、大小、奇偶、质合、012 路，一行看全。
- **底部统计**：出现次数、最大遗漏、当前遗漏。
- **期数切换**：近 30 / 50 / 100 / 全部。
- **新增开奖**：手动录入一期号码。
- **导入 / 导出 CSV**：更新或备份数据。
- **尝试在线更新**：尽力从官方接口抓取（多数情况会被浏览器跨域限制拦截，失败时请用导入 CSV）。

> 你新增、导入或在线更新得到的数据，**只保存在你这台设备的浏览器里**（localStorage），不会上传，也不会覆盖仓库里的内置示例数据。

## 关于数据

仓库自带的 `data/draws.json` 是**示例数据**（随机生成，仅用于展示走势图的样子）。
要看真实走势，把真实开奖号码导入进去即可，两种方式：

1. **导入 CSV**（推荐）：更多 → 导入 CSV，文件格式见 `data/sample-import.csv`：
   ```
   issue,date,number
   2026159,2026-06-16,480
   2026160,2026-06-17,725
   ```
2. **直接替换** `data/draws.json` 里的 `draws` 数组（每条 `{ "issue": "期号", "date": "YYYY-MM-DD", "n": [百,十,个] }`），然后重新提交到 GitHub。

## 部署到 GitHub Pages

1. 新建一个仓库，把这个文件夹里的内容（`index.html`、`styles.css`、`app.js`、`data/`、`.nojekyll`）全部放到仓库根目录。
2. 仓库 **Settings → Pages → Build and deployment → Source** 选 `Deploy from a branch`，分支选 `main`、目录选 `/ (root)`，保存。
3. 等一两分钟，页面会发布在 `https://<你的用户名>.github.io/<仓库名>/`。

### 绑定自己的域名

1. 在仓库根目录新建一个名为 `CNAME` 的文件，内容写你的域名，例如：
   ```
   fucai3d.example.com
   ```
2. 到你的域名服务商处加一条 DNS 解析：
   - 子域名（如 `fucai3d.example.com`）：加一条 `CNAME` 记录指向 `<你的用户名>.github.io`。
   - 裸域名（如 `example.com`）：加 `A` 记录指向 GitHub Pages 的 IP（`185.199.108.153` / `185.199.109.153` / `185.199.110.153` / `185.199.111.153`）。
3. 回到 **Settings → Pages**，在 Custom domain 填入域名，勾选 **Enforce HTTPS**。

## 本地预览

```bash
cd fucai3d-trend
python3 -m http.server 4179
# 浏览器打开 http://127.0.0.1:4179/
```

> 直接双击 `index.html`（file://）也能用：抓不到 `data/draws.json` 时会回退到内置的少量示例数据。用本地服务器可加载完整示例。
